  • Final Portfolio Summary

    The crime reporting application is a simple software that helps the public in the reporting of criminal activities. The general objective is to allow the public to report activates anonymously easily and in real-time. When an individual reports a criminal activity, the record is stored, and another person can view when a need arises. Once a reported crime activity is deleted, the application updates the records. The users can also view the reported crime activities, post, and comment on the reported criminal activities. Also, the users can chat with each other concerning the criminal activities reported and other relevant information. I have chosen this artefact as I partially created it in a prerequisite program to test my java skills.

    The enhancements area include the data structure and algorithms, database, and design part. Performing those enhance on the code will make it easier and reliable to correct available errors in the future (MacLeod et al., 2017).

    For the software design and engineering part of the project, There were partially developed IF-ELSE statements in several classes within the program. They lacked the default cases. Besides, there were wrongly arranged TRY-CASH classes that lacked the appropriate codes within the CATCH block. For instance, some blocks were incomplete, and some were even blank. Although those errors do not hinder the application program from executing, they often result in logical errors during run time. In the catch clause, it is challenging to realize if a certain error takes place as there is no default case or message in the mentioned parts and appropriate messages in the targeted catch blocks (Heckman& King, 2018).

    For the algorithms and data structures portion of the project, I delineated some of the weaknesses resulting from using unsuitable data structures and algorithms on the data structure and algorithms. Firstly, the crime report application freezes when a user posts a criminal activity. It resulted from the large amount of data retrieved from the database at the beginning and loaded on the main view module. Furthermore, the application lacked an appropriate algorithm applied in the project to create search forms for the crime activities' details.

    To improve the project, I rectified the initial weakness relating to application freezing at startup. To enable useful functionality, I developed an Array List to hold the data from the data. The data being stored is further used to create the only amount of the forms required to be displayed in the main module (Wei et al., 2020). The forms are held in a VBox layout where a scroll pane is utilized to hold the layout to enable scrolling to allow users to see all forms. At the end of the first form batch, I added a button to add another similar amount of forms of each data in the Array List. Since only ten forms are retrieved at startup, the freezing of the application was eliminated.

    On the database enhancement, the first weakness is that the crime report application database connection mechanism. To connect the crime application to the database, it included a different language like PHP. When using different programming language, it makes the connection complex and incompatible with its functions. Another weakness is that the database was hosted locally, which limited the accessibility to various users of the crime application.

    To improve the software, I corrected the initial weakness by using the popular Java Database connectivity approach (JDBC), which is easy and utilizes a single programing language; java. Using the JDBC approach, the connection becomes simple with a single file that holds the java connection codes rather than multiple files of different programming language codes (Tribhuvan, 2020). The other improvement involved hosting the database into a remote server, which enables multiple access by various users. Also, through remote server hosting, data retrieval and data transmission are smooth and take less execution time, unlike the local server hosting technique.

    In conclusion, those enhancements on data structure and algorithms, database, and software design have enabled the artefact to satisfy user requirements and meet its objectives completely. The application now enables universal database accessibility and takes less execution time to load and display crime activity details, unlike the previous application. Data searching algorithm is now quick and easy. The application currently does not freeze at users' crime activity posting.
